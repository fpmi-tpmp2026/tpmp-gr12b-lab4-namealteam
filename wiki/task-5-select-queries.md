# Задание 5. SELECT-запросы и отчёты

## Что реализовано

В приложении туристического бюро SELECT-запросы собраны в модуле [`src/reports.c`](https://github.com/namealx/lab4-tourism/blob/main/src/reports.c) и доступны из меню администратора.

### Отчёты

| Функция | Назначение |
| --- | --- |
| `report_bus_trips_period` | Вывод рейсов автобуса за выбранный период |
| `report_bus_summary` | Статистика автобуса за период: количество рейсов, пассажиры, выручка |
| `report_crew_salary_period` | Просмотр начислений экипажам за период |
| `report_most_expensive_route` | Поиск самого дорогого маршрута и связанных рейсов |
| `report_bus_with_max_mileage` | Поиск автобуса с наибольшим пробегом |

## SQL-логика

В отчётах используются:

- `JOIN` между `trips`, `routes`, `crew` и `buses`;
- агрегатные функции `COUNT`, `SUM`, `COALESCE`;
- фильтрация по диапазону дат через `date(...) BETWEEN ...`;
- сортировка по дате, названию маршрута и номеру автобуса.

## Как проверить

1. Запустить приложение и войти под `admin`.
2. Открыть меню `Отчёты и выборки`.
3. Выбрать один из пунктов 1-6.

## Вывод

SELECT-запросы работают в связке с данными из SQLite и возвращают отчёты, которые можно использовать в административном меню и в отчёте по лабораторной работе.
