# Функциональные требования

## Используемые случаи

1. **Регистрация пользователя**  
   **Акторы:** Новые пользователи  
   **Предусловия:** Пользователь должен иметь доступ к интернету.  
   **Сценарий:**  
   - Пользователь открывает страницу регистрации.  
   - Пользователь вводит свои данные (имя, email, пароль).  
   - Пользователь нажимает кнопку "Зарегистрироваться".  
   - Система проверяет данные и создает новую учетную запись.  
   - Пользователь получает сообщение об успешной регистрации.  

2. **Вход в систему**  
   **Акторы:** Зарегистрированные пользователи  
   **Предусловия:** Пользователь должен быть зарегистрирован.  
   **Сценарий:**  
   - Пользователь открывает страницу входа.  
   - Пользователь вводит свои учетные данные (email, пароль).  
   - Пользователь нажимает кнопку "Войти".  
   - Система проверяет данные и перенаправляет пользователя на главную страницу.  
   - Пользователь получает сообщение об успешном входе.  

3. **Сброс пароля**  
   **��кторы:** Пользователи, которые забыли пароль  
   **Предусловия:** Пользователь должен иметь доступ к электронной почте.  
   **Сценарий:**  
   - Пользователь открывает страницу сброса пароля.  
   - Пользователь вводит свой email.  
   - Пользователь нажимает кнопку "Отправить ссылку для сброса".  
   - Система отправляет на email ссылку для сброса пароля.  
   - Пользователь получает сообщение об успешной отправке.  

## Сценарии

- **Сценарий 1:** Пользователь успешно регистрируется.  
- **Сценарий 2:** Пользователь не может войти в систему из-за неверного пароля.  
- **Сценарий 3:** Пользователь получает ссылку для сброса пароля на указанный email.
